微服务DEMO编写
背景
SpringCloud 是 Pivotal 团队提供的一整套微服务开源解决方案，包括服务注册与发现、配置中心、全链路监控、服务网关、负载均衡、断路器等组件。
需求
一、 用一个 docker-compose.yaml 启动多个服务：
1. Uaa 用户认证 [数据库里的用户名密码登录，GITHUB oauth2登录， LDAP登录]
2. MySQL
3. LDAP服务
4. Gateway  [仅此服务对外开放端口7573，不用HTTPS]
5. Configure
6. Discovery
7. Product [产品表可以只包含自增主键+产品名]
   如果您的组件选型中，有些组件其设计本身就可以承担多个服务的角色，不需要严格使用上面列出的服务。
   浏览产品列表API需要角色USER，
   添加产品API：需要角色 EDITOR,
   修改产品API：需要角色 EDITOR
   删除产品API：需要角色 EDITOR
   角色: PRODUCT_ADMIN 拥有EDITOR， USER所有权限; EDITOR 拥有USER所有权限。
   二、用Springboot cloud[最新稳定版] + maven + Java 实现服务
